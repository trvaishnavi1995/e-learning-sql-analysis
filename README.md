# E-Learning Platform Analysis using MySQL

## Project Overview

This project analyzes purchase data from an e-learning platform using MySQL to uncover insights about learner behavior, course performance, and revenue trends.

---

## Objectives

* Analyze learner purchasing behavior
* Identify top-performing courses and categories
* Calculate revenue trends
* Detect high-value learners
* Identify courses with no purchases

---

## Database Design

The project uses a relational database with three tables:

### 1. Learners

* learner_id (Primary Key)
* full_name
* country

### 2. Courses

* course_id (Primary Key)
* course_name
* category
* unit_price

### 3. Purchases

* purchase_id (Primary Key)
* learner_id (Foreign Key)
* course_id (Foreign Key)
* quantity
* purchase_date

---

##  SQL Concepts Used

* Joins (INNER JOIN, LEFT JOIN, RIGHT JOIN)
* Aggregations (SUM, COUNT)
* Grouping (GROUP BY, HAVING)
* Sorting & Filtering (ORDER BY, WHERE, LIMIT)
* Derived calculations (Revenue = quantity × price)

---

## Key Analysis Performed

### Total Spending per Learner

Identifies high-value customers based on total purchase amount.

### Top 3 Most Purchased Courses

Finds the most popular courses based on quantity sold.

### Category-wise Revenue

Analyzes revenue generated by each category.

### Multi-Category Learners

Identifies learners engaged across multiple categories.

### Unpurchased Courses

Highlights courses with zero purchases.

---

## Key Insights

* Programming and AI categories generate high revenue
* SQL and Excel courses are highly popular
* Some learners contribute significantly to revenue
* Certain courses have no demand

---

## Recommendations

* Focus marketing on high-performing categories
* Promote low-performing courses
* Offer bundled courses
* Target high-value learners
